### .rm v6 Parser Rewrite (rmscene format)

- Completely rewrote `rm-file-parser.ts` based on binary analysis of actual .rm files
- **Block header format** (8 bytes): `uint32 length | uint8 unknown(0) | uint8 min_ver | uint8 cur_ver | uint8 type` — previous parser had wrong order (type first, varints for versions)
- **Block types** corrected: SceneLineItemBlock=0x05, AuthorIdsBlock=0x09, PageInfoBlock=0x0A, SceneInfoBlock=0x0D, etc.
- **Tagged value format**: `(index << 4) | wire_type` as varuint. Wire types: ID=0xF (CrdtId), Length4=0xC, Byte8=0x8, Byte4=0x4, Byte1=0x1
- **CrdtId**: uint8 (author) + varuint (counter)
- **SceneLineItemBlock structure**: CRDT item header (tags 1-6) → value subblock → scene type byte (3=Line) → tagged line fields (tool, color, thickness, starting_length, points subblock)
- **Point format v2** (14 bytes): `float32 x, y | uint16 speed, width | uint8 direction, pressure`
- Added `TagType`, `SceneItemType`, `BLOCK_HEADER_SIZE` constants to `rm-constants.ts`
- Verified working with 6 notebooks (1-15 pages each), zero parse errors

## Key Decisions

- Token storage at ~/.remarkable-sync/token.json (outside vault)
- Desktop-only (OffscreenCanvas, filesystem tokens)
- Sync v3 protocol (`internal.cloud.remarkable.com/sync/v3/`) for document listing and download (v1 API deprecated, returns 404)
- Removed all transcription features (Ollama, TrOCR, line-segmenter) — plugin focused on list/download/sync
- Sync state stored as `syncStore` property within `PluginSettings` (single `saveData` object)
- .rm v6 format uses rmscene tagged block format (not simple block type + varint versions)
